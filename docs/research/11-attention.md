# Research note 11 — Attention ordering: the scenario-weight firewall's positive half

Review slice S-C (SPEC-22) · per ASSAY-DEC-11 · 2026-07-16 · bounded to hours, one page preferred
Prompts (review `docs/reviews/2026-07-14-jipoe-c2-process-review.md` §3.5, mockup M11, addendum slice S-C): the register's firewall language says scenario weights "**order attention and reporting**, and never compile into a constraint or cost" (knowledge model §9). The negative half is implemented and test-asserted (`encoding.ts` `mayCompileAsConstraintOrCost`, the SPEC-06 compile partition, note 02 §2). The positive half has no visible implementation: no surface renders the K14a–c likelihood bands and nothing orders by them. This note decides the honest ordering rule for overlapping bands and the exact queue behaviours. Output feeds SPEC-22 (`specs/022-attention-ordering/`).

## 1. Why the positive half must exist, doctrinally

JP 2-01.3 (JIPOE step 4, §II-9) *requires* a likelihood rank-ordering of the adversary COA set — identifying the most-likely and most-dangerous COAs is a J-2 product, not an optional flourish. ICD 203 governs how that judgement is expressed (words of estimative probability, mapped to percentage ranges — exactly the authoring rule note 01 §"Likelihood bands" already applies to K14a–c). ASSAY's design answer is already the honest one: the ordering **exists** (K14a–c, banded, ICD 203-ranged), is **contestable** (normal lifecycle), and influences **attention, never arithmetic** (the §9 firewall). But a demonstrator that holds likelihoods and does nothing visible with them will read to an SME as a gap, not a discipline. The positive half is the missing exhibit: likelihood doing its one legitimate job, and only that job.

## 2. The ordering rule, decided: the interval order — nothing else

Ordering bands is exactly where a midpoint sneaks back in. The candidate rules, and why all but one fail:

- **Any scalar sort key** — sort by `lo`, by `hi`, by centre, by width-penalised centre: every one collapses the band to a point and manufactures a total order the assessment does not support. Sorting R2 (20–40%) above R3 (10–25%) by *any* scalar asserts a ranking the J-2 never made. Rejected by DEC-15 (a Band is a pure interval with no interior) and DEC-19 (no derived scalar in decision arithmetic — and an ordering *is* arithmetic when it feeds a queue).
- **Stochastic orders** (first-order stochastic dominance and kin) — require a distribution over the band's interior. A Band has no interior (DEC-15; the same ground on which note 10 rejected HOPs and quantile dotplots). Rejected.
- **Hurwicz-style mixtures** (α·lo + (1−α)·hi) — a weighted blend with a smuggled α. Rejected by DEC-19; α is precisely the authored scalar the register forbids.
- **The interval order** (Fishburn): `R_a` ranks strictly above `R_b` **iff `lo(R_a) > hi(R_b)`** — the whole of one band sits above the whole of the other. Overlapping bands (nesting included — nesting is overlap) are **honestly incomparable**. This is already the conservative comparison the dominance machinery uses (`dominance.ts` `strictlyBetter` with ε = 0; note 03 §5), so the positive half reuses the machinery the register has already blessed — **no new comparison machinery**.

**Stratification into attention layers.** A strict partial order renders as levels: scenario `s` is placed in layer `1 + (length of the longest chain of strictly-above relations above s)` — layer 1 holds the scenarios nothing ranks above; scenarios the order cannot separate render **level**. A scenario renders below another *iff* the interval order actually says so; incomparability is never resolved by presentation. On Meridian this produces the exhibit the firewall deserves: **R1 (45–70%) strictly above the level pair {R2 (20–40%), R3 (10–25%)}** — `lo(R1)=45 > hi(R2)=40` and `45 > hi(R3)=25`, while `lo(R2)=20 ≤ hi(R3)=25` leaves R2/R3 visibly unranked. The system refuses, in public, to manufacture the R2-vs-R3 ranking the bands don't support. Within a layer, scenarios render in logical-id order — a **stated presentation order**, carrying no likelihood claim (the same latitude note 06 §4 spent on the verdict grid's scenario columns).

**Edge cases, decided:**
- **All bands overlapping** → one layer, all level. "We cannot rank these" is itself the exhibit.
- **No likelihood object** for a scenario → renders **unranked** ("no assessment"), never defaulted to a uniform weight — absence of judgement is not a judgement.
- **Contested or superseded K14** → normal lifecycle. A contested weight renders its band with the contest mark but joins the unranked group and **drops out of queue tie-breaking** until resolved: a disputed assessment orders nothing (it never compiled anyway). A superseded weight's successor orders; the predecessor does not.

## 3. The queue tie-break, decided: weight breaks ties only, and says so

The S1 collection queue's primary ranking is discrimination separation (DEC-18, SPEC-12) and is **never overridden** by likelihood. The one legitimate *process* use of a scenario weight is directing scarce collection attention between questions the primary ranking cannot separate:

1. **Equal standing** means the best-separation bands are identical — `lo` *and* `hi` exactly equal. Band equality, not band overlap: a tolerance would be a new scalar.
2. **The pair a question bears on** is its best-separating COA pair (the pair achieving `best_separation`; if several tie, all of them).
3. **The pair lift of the interval order**: pair `P` ranks above pair `Q` iff every scenario in `P \ Q` is strictly above (interval order, §2) every scenario in `Q \ P`. Shared scenarios cancel; an empty symmetric difference means the same pair, so no tie-break. Where a question has several best pairs, every pair of one question must rank above every pair of the other — universal, conservative, no manufactured ranking. On Meridian K14: {R1,R2} ranks above {R2,R3} (R1 strictly above R3), but {R1,R2} vs {R1,R3} is unranked (R2/R3 overlap).
4. **Where the lift ranks the pairs**, the question bearing on the higher pair orders first **and the tie-break is stated in the rendering** — "tie broken by scenario weight (attention only)" — DEC-19 style: stated, never silent. **Where it does not rank** (overlapping likelihoods, missing or contested weights), no tie-break applies and the existing stated fallback order holds (the queue's stable order — the request's question order).
5. **Placement**: the tie-break lives in the **queue assembly** (the app-state layer that composes the S1 queue), not in `DiscriminationService`. The analysis service stays weight-free — its ranking, its stamp, and DEC-18 are untouched; the attention layer composes *around* it. This is what keeps "attention only" machine-checkable (§4).

## 4. "Attention only", machine-checkably

The register phrase becomes two testable properties:

- **Zero computational movement**: editing a K14 band (a legal knowledge write — `checkEncoding` passes `scenario_weight`; the firewall is at compile) changes **no** world stamp, verdict, score, handful membership, relaxation output, or analysis stamp. Same stamps and seeds before and after; only strip emphasis and queue tie-break annotations may move.
- **Import isolation**: the attention module and the strip import no scoring machinery (`compile`/`score`/`materialise`/`metrics`/`relax`/`handful`), and the compile/score/dominance/relax code imports nothing from them. The blessed reuse is `dominance.ts`'s comparison — attention *reads* the comparison; the scorer never reads attention.

The two behaviours of §2–§3 — (a) the strip's banded likelihood ordering, (b) the queue's stated tie-break — are the **complete** set the sharpened register language licenses. Any third weight-consuming behaviour is a new register candidate, not an extension.

## 5. The surface

The scenario strip (SPEC-10) gains an **attention block** above the verdict grid: one line per adversary COA — id, name, and the likelihood band rendered as a `bandPill` on a **shared 0–100% track** so overlap is visible as geometry, grouped into layer rows (strictly-ranked layers stacked, level scenarios side by side in one row). Provenance is welded on, not hoverable-optional (DEC-9): each line carries the provenance chip ("assessed · moderate — assessment, not fact", owner J-2 fusion) and the standard `data-logical-id` hook so the one-hop trace opens back to the owner (G3). The block carries a **persistent label — "orders attention — never compiles"** — keyed in the per-component legend. The verdict grid to its right is untouched: same cells, same scenario-column order (logical id, note 06 §4) — likelihood moves nothing but position and emphasis inside the attention block. No scalar sort key exists in the DOM or the code: ordering is pairwise interval comparison, rendered structurally as layers.

## What we will do differently

1. **Adopt the interval order as the one honest ordering for likelihood bands** — `lo(a) > hi(b)` or unranked — rejecting every scalar sort key (DEC-15/19), stochastic orders (no distribution exists), and Hurwicz mixtures (a smuggled α). Reuse `dominance.ts`'s comparison; build no new comparison machinery.
2. **Render the ordering as attention layers**: longest-chain stratification, incomparables level, unranked (missing/contested) scenarios explicitly "no assessment" — never uniform-defaulted. Meridian's exhibit: R1 strictly above a visibly overlapping, level {R2, R3}.
3. **The S1 queue tie-breaks on scenario weight only at exact discrimination equality**, via the conservative pair lift of the interval order, stated in the rendering wherever applied, never applied across overlap, never overriding DEC-18's primary ranking.
4. **Keep the analysis services weight-free**: the tie-break composes in the queue assembly; `DiscriminationService` and its stamp are untouched.
5. **Pin both halves**: a weight edit changes zero stamps while the presentation moves (the machine-checkable "attention only"), and import isolation keeps the firewall structural, not conventional.

**Register candidate** (concept §6.24, flagged — DEC-2): sharpen "order attention and reporting" into exactly the two named behaviours above; any third use is a new candidate. This note asserts no register decision: the ordering rule is forced by DEC-15/19 once a visible ordering is required at all; the queue placement is DEC-18 respected; the surface rules are G2/G3/DEC-9 applied. Latitude spent (layer stratification by longest chain, logical-id order within a layer, band-equality as the tie-break trigger, the universal pair lift) is Stage-delegated presentation/process latitude under the review-slice remit and is recorded here, not asserted in a peer document.
