# Implementation Plan: Compile — knowledge → CompiledWorld (SPEC-06)

**Branch**: `claude/next-spec-gfmn70` (spec dir `006-compile`) | **Date**: 2026-07-12 | **Spec**: [spec.md](./spec.md)

**Input**: Feature specification from `specs/006-compile/spec.md`

## Summary

Add the `/compile` seam service — knowledge subset + vignette config → CompiledWorld — over the existing store (SPEC-01), trace store (SPEC-02), and knowledge service (SPEC-05). Channels are stored **sparse** (`default: Band` + named, time-boxed `RegionOverride`s), retiring dense `Channel.cells` per research note `02-compile.md`; a new `VignetteConfig` holds grid, per-channel defaults, region geometry, and the `subject → (channel, region)` map. The compile refuses `contested_knowledge` (reusing SPEC-05's `isCompilable` — completing G5 at the compile refusal), `stale_input` (via `effectiveStatus`), and `encoding_violation`/`waiver_required` (defence-in-depth via the pure `checkEncoding`); it firewalls `scenario_weight`; it applies a COA excursion as a region-override layer folded into the stamp. The stamp is a hash over consumed refs + config + engine version (G1). One `compiled_into` edge per consumed object plus a `waives` edge per waiver make every channel value backward-traceable to named owners (G3); each `RegionOverride` also names its `source`. A framework-free compile refusal banner and channel-trace surface render the demo moment. Scope excludes the scorer and generator (SPEC-07/08) and the routing-search structure (Stage 3).

## Technical Context

**Language/Version**: TypeScript (strict, ESM, `.js` import specifiers), in-browser + Node ≥ 19 for tests (WebCrypto one code path — research note 00).

**Primary Dependencies**: none at runtime (offline); dev: `vitest`, `tsx`, `tsc`, LinkML type-gen (`npm run gen`). Reuses `src/{store,trace,canonical,validate,seam,encoding,knowledge}.ts`, `src/generated/types.ts`, `src/components/{bandPill,provenanceChip,refusalBanner}.ts`.

**Storage**: in-memory content-addressed `ObjectStore` + `TraceStore` (existing). No new persistence; the CompiledWorld and VignetteConfig are ordinary stored objects.

**Testing**: `vitest` (`npm test`). Contract-invariant tests for compile written against the seam §4 shape and confirmed failing before implementation (constitution quality gate 2). Stamp-determinism and G2/G3/G5 tests are Stage-2 acceptance.

**Target Platform**: browser (offline SPA) and Node test runner; single code path.

**Project Type**: single project (in-browser demonstrator behind a service seam).

**Performance Goals**: a compiled Meridian world canonicalises + hashes in well under one second (sparse channels); the ~19.4 s dense cost (measured, `npm run bench`) is the failure mode this representation exists to avoid (SC-006). Determinism (G1) is the hard requirement.

**Constraints**: offline-capable; LinkML is the source of truth (types generated by `npm run gen`, never hand-drifted — this slice changes the schema then regenerates); surfaces arrange projections only; banded honesty enforced at the seam and on the surface; **no dense per-cell channel is ever stored or hashed**.

**Scale/Scope**: 6 channels, ~a dozen consumed objects, tens of region overrides (not 1.2M cells); one compile service, one schema change + regen, one new config fixture, two surfaces (refusal banner reuse + channel-trace).

## Constitution Check

*GATE: evaluated before Phase 0 and re-checked after Phase 1 design.*

**Principle I — Seam Contract Is the Invariant.** PASS. The service implements seam §4 exactly (`{knowledge, scenario?, config, engine_version} → {world, stamp, compiled_from} | Refusal`); refusals are first-class returns, not HTTP errors (seam §1); the channel-trace surface arranges projections of the compile response and holds no private derived state (FR-015).

**Principle II — Banded Honesty (NON-NEGOTIABLE).** PASS. Every channel value is a `Band` with provenance reachable via `source` (FR-013); only `observed`-derived values may be degenerate; no bare assessed scalar crosses the seam or renders (SC-007). The sparse representation stores bands, never bare cell scalars. **(G2)**

**Principle III — Traceability Terminates in Named Knowledge.** PASS and central. One `compiled_into` edge per consumed object and a `waives` edge per waiver are written at compile time by the compiling service (FR-011); every `RegionOverride` names its `source` (FR-012); a backward walk terminates in named owners with no dead ends (SC-003). The compile firewall (contested/stale/encoding) is enforced here as part of this principle. **(G3, G5)**

**Principle IV — Determinism and Content Addressing.** PASS. The stamp is a hash over sorted consumed refs + config + engine version + optional seed (FR-010); identical inputs yield byte-identical stamp and world; no ambient entropy; wall clock never participates (DEC-17). The world is immutable and content-addressed; recompiling identical inputs is idempotent at the store. **(G1)**

**Principle V — Research Before Implementation.** PASS. `docs/research/02-compile.md` exists (the DEC-11 gate) and decides the sparse representation, the excursion-as-overlay model, the time-windowing, and the refusal reuse this plan implements.

**Principle VI — Register Supremacy.** PASS. Every FR cites the register DEC or invariant it restates; the one new decision (sparse channels retiring `Channel.cells`) is recorded as a flagged register candidate (concept §6, item 12) and built under seam open item 2's delegated authority, not asserted here.

**Invariant applicability at Stage 2**: G1 (stamp determinism — central exit), G2 (banded channel values), G3 (compiled_into edges + `source`, backward-complete), G5 (contested never compiles — **completed** here at the compile refusal; SPEC-05 supplied the mark+block). G4 (least-worst) and G6 (propagation honesty) are **not exercised** at Stage 2 — no relaxation, no interval arithmetic in the compile (channel values are the authored bands, not propagated products) — and are asserted at SPEC-09 and SPEC-07/oracle cases respectively.

**Gate result**: PASS, no violations. Complexity Tracking table omitted (nothing to justify).

## Approach (how the pieces fit)

- **Schema first, then regen.** Edit `schema/*.yaml`: replace `Channel.cells` with `default: Band` + `regions: RegionOverride[]`; add `RegionOverride`, `VignetteConfig`, `ChannelDefault`, `RegionGeometry`, `SubjectMapEntry`; remove `ChannelCell`. Run `npm run gen`; the bench script (`scripts/bench-canonical.ts`) is dense-only and stays as the documented counterfactual — it is not part of the compile path.
- **Refusal before construction.** `compile.ts` resolves refs, then runs the three refusal gates (`contested_knowledge` via `isCompilable`, `stale_input` via `effectiveStatus`, encoding via `checkEncoding`) and `unknown_ref` — computing any refusal *before* building or storing anything, so a refused compile persists nothing (FR-002 analogue; SC-001).
- **Partition then build.** Consumed objects partition into firewalled (`scenario_weight` — never a channel/edge), skipped (answer absent — open questions), and consumable. Each consumable routes through `config.subject_map` to a `(channel, region)`; its answer becomes a `RegionOverride{value, from/until from validity, source}`. Per-channel defaults come from the config.
- **Excursion as overlay.** If `scenario` is supplied, the `ScenarioCOA.excursion` entries are applied as additional `RegionOverride`s and folded into the stamp's config — one shape for base and variant, scorer stays scenario-blind (DEC-10).
- **Stamp + edges.** `stamp = contentHash({consumed: sortByLogicalId(refs), config, engine_version, seed?})`. Store the world (idempotent). Write one `compiled_into` edge per consumed object and a `waives` edge per waiver-carrying object. No delta (D7).
- **Surface.** `channelTrace.ts` renders each channel's regions: band pill + provenance + backing KnowledgeObject (the `source`), a projection of the compile response. The compile refusal reuses `refusalBanner.ts`, with the contested pair rendered side by side (the S1 contested rendering from SPEC-05).
